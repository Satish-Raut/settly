## 2. Concrete Cases of AI Errors & Corrections

To show the interviewer that I remained the **engineer of record** and audited every line of code generated by the AI, here are three specific cases where the AI produced incorrect logic and how I corrected it:

### Case 1: Incorrect Date Parsing Logic (MM/DD/YYYY Mismatch)
* **The AI's Mistake**: The AI generated a date normalization helper that assumed all date formats in the CSV were standard `DD-MM-YYYY`. However, Row 34 in our CSV: `04-05-2026,Deep cleaning service...` was parsed by the AI as April 5th, ignoring the note: *"is this April 5 or May 4? format is a mess"*.
* **How I Caught It**: When comparing the output against the physical CSV notes, I noticed the ambiguity flag was bypassed, causing a silent incorrect date entry of April 5th.
* **What I Changed**: I modified the parser function to look for double-digit ambiguity (values $\le 12$ for both date components) and force a `CSVAnomaly` flag of type `AMBIGUOUS_DATE_WARNING`, which halts the row from importing until the user explicitly selects DD/MM or MM/DD in the staging wizard.

---

### Case 2: Rounding Precision Loss in Equal Splits
* **The AI's Mistake**: In the equal split logic, the AI generated:
  ```javascript
  const share = Math.round((amount / count) * 100) / 100;
  ```
  While this rounded individual shares to 2 decimals, it introduced a division remainder error:
  * For Row 10: Cylinder Refill `899.995` split equally among 4 users.
  * `899.995 / 4` $\to$ `224.99875` $\to$ Rounded to `225.00`.
  * `225.00 * 4 = 900.00`. The sum of splits ($900.00$) exceeded the actual total ($899.995$).
* **How I Caught It**: During unit testing of `splitService.test.ts`, the total sum of allocated splits did not match the original expense amount, causing a data integrity warning.
* **What I Changed**: I added a remainder redistribution algorithm. The logic now calculates the difference between the actual total amount and the sum of individual rounded splits:
  ```javascript
  const remainder = amount - (roundedShare * count);
  // Assign the remainder to the first split record
  splits[0].shareAmount += remainder;
  ```
  This guarantees that the sum of splits matches the total amount to the exact cent, preventing rounding leakage.

---

### Case 3: Staging File Path Reference Error
* **The AI's Mistake**: When configuring artifact directories, the AI initially attempted to write local system configuration files to directories outside of the active workspace, which triggered system permission path errors.
* **How I Caught It**: The compiler/IDE logs threw path validation exceptions when trying to read configuration data from a relative nested folder.
* **What I Changed**: I updated the file paths to enforce absolute paths pointing to the project directory: `d:\WebDev\Projects\SPREETAIL Project\...` to ensure the project structures remain unified and containerizable.
